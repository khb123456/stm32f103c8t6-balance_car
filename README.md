# STM32F103C8T6 两轮自平衡小车项目
> Two-Wheeled Self-Balancing Vehicle Based on STM32F103C8T6
> （开发中 / Under Development）

---

## 🚀 项目简介
本项目基于 **STM32F103C8T6** 单片机，实现两轮自平衡小车的**直立控制、速度闭环控制、方向控制**，采用**串级PID**控制算法。
搭配 MPU6050 姿态传感器、编码器测速、TB6612 电机驱动，实现稳定的自平衡控制。

---

## 🛠️ 硬件清单
- 主控：STM32F103C8T6 最小系统板
- 姿态传感器：MPU6050（六轴陀螺仪+加速度计）
- 电机驱动：TB6612 直流电机驱动模块
- 电机：7.4V 直流减速电机（带正交编码器）
- 电源：12V 锂电池 / 12V 电源适配器
- 显示模块：0.96寸 OLED（I2C）
- 通信模块：JDY-31 蓝牙模块
- 测距模块：HC-SR04 超声波模块
- 电源模块：5V 降压模块
- 结构件：轮趣 R1 平衡车底盘套件

## 🧾 项目问题与解决记录
### 1. OLED 软件I2C无显示问题
- **问题**：硬件I2C正常，软件I2C模式下OLED不亮
- **原因**：软件I2C时序不严格，缺少必要延时导致通信失败
- **解决**：使用逻辑分析仪抓取时序，添加精准延时，修复通信时序

### 2. MPU6050 数据读取过慢
- **问题**：单次读取耗时 20ms+，无法满足平衡控制实时性
- **原因**：采用单字节读取，代码冗余、延时过多
- **解决**：改为**多字节连续读取**，统一解析数据，删除无效延时
- **效果**：读取速度从 20ms → **2ms 内**

### 3. 左轮编码器无法读取
- **问题**：编码器突然无数据，代码无错误
- **排查**：逻辑分析仪测量AB相波形，电机本身正常
- **解决**：更换扩展板后恢复正常
- **结论**：硬件接线/扩展板接触不良导致

### 4. Vofa+ 调试助手无数据显示
- **问题**：其他串口工具正常，Vofa+ 不显示
- **原因**：Vofa+ 要求数据结尾必须带 `\r\n` 换行符
- **解决**：发送数据添加 `\r\n` 格式，正常收发显示
